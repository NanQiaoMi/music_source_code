## 2. 分阶段实施计划

### 阶段一: 集成验证与基础修复（P0）

**目标**: 确认 28 个效果能在实际播放中正确渲染

| 任务                 | 描述                                                | 验收标准                                   |
| -------------------- | --------------------------------------------------- | ------------------------------------------ |
| 1.1 效果清单盘点     | 运行每个效果，记录工作状态                          | 列出所有效果及其状态（可用/需修复/未实现） |
| 1.2 音频数据管道检查 | 确认效果能从 AudioEngine 获取实时频谱/波形/节拍数据 | 每个效果都有音频响应                       |
| 1.3 渲染循环整合     | 确认 RenderEngineManager 正确驱动所有效果           | 切换效果流畅，无闪烁                       |
| 1.4 基础 bug 修复    | 修复集成测试中发现的问题                            | 核心效果可用                               |
| 1.5 效果切换稳定性   | 确保切换效果时无内存泄漏、无闪烁                    | 连续切换 30 次无异常                       |

**预计工作量**: 2-3 天

### 阶段二: 性能体系建设（P1）

**目标**: 建立完整的性能管理，确保低端/高端设备各有合适配置

| 任务             | 描述                            | 验收标准                                 |
| ---------------- | ------------------------------- | ---------------------------------------- |
| 2.1 性能档位定义 | 低/中/高/极致 四档参数          | 每档有明确的渲染质量/分辨率/粒子数量定义 |
| 2.2 自动检测     | 启动时检测 GPU/内存自动推荐档位 | 自动选择匹配档位                         |
| 2.3 实时性能监控 | FPS 监控 + 自动降级             | FPS < 30 时自动降级                      |
| 2.4 对象池化     | 粒子/几何体对象池，避免 GC 抖动 | 连续运行 30min 内存无增长                |
| 2.5 手动切换     | 用户可手动覆盖性能档位          | UI 中有性能设置入口                      |

**预计工作量**: 2-3 天

### 阶段三: 参数系统完善（P1）

**目标**: 用户可通过面板调节效果参数

| 任务               | 描述                       | 验收标准                        |
| ------------------ | -------------------------- | ------------------------------- |
| 3.1 效果参数定义   | 每个效果定义可调节参数列表 | 参数字段名/类型/范围/默认值完整 |
| 3.2 参数 UI 面板   | 滑块、颜色选择器、下拉菜单 | 所有参数类型都有对应 UI 控件    |
| 3.3 参数与效果绑定 | 调节参数即时响应           | 滑块拖动时效果实时变化          |
| 3.4 参数持久化     | 关闭页面后恢复参数         | 刷新页面参数保持                |

**预计工作量**: 2 天

### 阶段四: 预设管理与异常处理（P2）

**目标**: 用户可保存/加载效果预设，系统稳定可靠

| 任务                      | 描述                        | 验收标准               |
| ------------------------- | --------------------------- | ---------------------- |
| 4.1 预设存取              | 保存当前全部参数为预设      | 预设可命名、保存、删除 |
| 4.2 预设导入导出          | JSON 文件导入导出           | 可分享预设文件         |
| 4.3 内置预设              | 提供 5-10 个官方预设        | 覆盖主要效果类型       |
| 4.4 Canvas 上下文丢失处理 | WebGL context lost 自动恢复 | 恢复后效果继续运行     |
| 4.5 渲染异常捕获          | 单个效果崩溃不拖垮整体      | 错误效果自动跳过并提示 |

**预计工作量**: 2 天

## 3. 核心接口定义

### 效果插件接口

```typescript
interface EffectPlugin {
  id: string;
  name: string;
  category: EffectCategory;
  description: string;
  thumbnail?: string;

  preferredEngine: "canvas" | "webgl" | "auto";

  parameters: EffectParameterDefinition[];

  init: (ctx: RenderContext) => void;
  render: (ctx: RenderContext, audioData: AudioData) => void;
  resize: (width: number, height: number) => void;
  destroy: () => void;
}
```

### 参数定义

```typescript
interface EffectParameterDefinition {
  id: string;
  name: string;
  type: "number" | "color" | "select" | "boolean";
  defaultValue: unknown;
  min?: number;
  max?: number;
  step?: number;
  options?: { label: string; value: unknown }[];
}
```

## 4. 技术风险与应对

| 风险                        | 影响         | 应对策略                              |
| --------------------------- | ------------ | ------------------------------------- |
| 某些效果在低端 GPU 崩溃     | 用户体验差   | 阶段二的自动降级 + Canvas 2D fallback |
| 28 个效果内存占用过大       | 页面卡顿     | 对象池 + 懒加载（仅当前效果加载）     |
| 音频数据与效果参数不匹配    | 效果响应迟钝 | 统一 AudioData 接口定义               |
| Three.js 与 Canvas 切换闪烁 | 视觉干扰     | 预加载 + 交叉渐变过渡                 |
