## 🎯 迭代目标与范围

### 本次迭代目标

1. **可视化界面**：实现基于 Three.js 的音频可视化效果
2. **音频增强**：Web Audio API 音频特效、A-B 循环、响度分析
3. **专业工具**：音频指纹、格式转换、CUE 分轨、音乐库健康检查
4. **桌面集成**：Electron 桌面歌词、系统托盘（并行开发）

### 本次迭代范围

| 模块 | 包含功能 | 优先级 |
|-----|---------|-------|
| **基础框架** | 分支管理、目录结构、uiStore 扩展 | P0 |
| **可视化界面** | 3-5 种核心效果、参数调节、预设系统 | P0 |
| **音频功能** | 音频特效、A-B 循环、录制功能 | P1 |
| **专业工具** | 音频指纹、格式转换、CUE 分轨 | P1 |
| **Electron** | 桌面歌词、系统托盘 | P2 |

### 明确不包含

- ❌ 现有三个核心界面的 UI 修改
- ❌ 现有组件的重构
- ❌ 技术栈的重大变更

## ⏰ 时间节点规划

### 阶段一：基础框架（1-2 天）
- **目标**：建立开发环境，验证核心机制
- **交付物**：分支、目录结构、uiStore 扩展、POC 原型
- **验收**：用户确认基础框架

### 阶段二：可视化界面（3-5 天）
- **目标**：实现核心可视化效果
- **交付物**：可视化组件、参数面板、预设系统
- **验收**：用户确认可视化功能

### 阶段三：音频功能增强（2-3 天）
- **目标**：实现音频特效和增强功能
- **交付物**：音频特效面板、A-B 循环、录制功能
- **验收**：用户确认音频功能

### 阶段四：专业工具面板（2-3 天）
- **目标**：实现专业音频处理工具
- **交付物**：音频指纹、格式转换、CUE 分轨
- **验收**：用户确认专业工具

### 阶段五：Electron 并行开发（3-4 天）
- **目标**：实现桌面集成功能
- **交付物**：Electron 框架、桌面歌词、系统托盘
- **验收**：用户确认桌面功能

### 阶段六：测试与优化（2-3 天）
- **目标**：完整测试和性能优化
- **交付物**：测试报告、优化后的代码、完善的文档
- **验收**：最终验收

## 🛠️ 技术栈与资源调配

### 前端技术栈（保持不变）

- **框架**：Next.js 14 + React 18
- **语言**：TypeScript
- **样式**：Tailwind CSS
- **状态管理**：Zustand
- **动画**：Framer Motion
- **图标**：Lucide React

### 新增技术

- **可视化**：Three.js、WebGL
- **音频**：Web Audio API、MediaRecorder
- **桌面**：Electron、IPC
- **音频处理**：Chromaprint WASM（可选）

### 依赖安装

```bash
# Three.js 可视化
npm install three @react-three/fiber @react-three/drei

# Electron（后续阶段）
npm install -D electron electron-builder
```

## 🔍 代码审查与测试策略

### 代码审查流程

1. **功能开发**：在独立分支开发
2. **自我检查**：确认未修改现有三个界面
3. **提交 PR**：提供功能描述和截图
4. **用户审查**：用户确认（特别确认现有 UI 未修改）
5. **合并代码**：确认后合并到 feature/v7.0

### 审查检查清单

- [ ] 现有三个核心界面的 UI 完全未被修改
- [ ] 现有组件只复用不修改
- [ ] 新增代码在正确的隔离目录中
- [ ] TypeScript 类型完整
- [ ] 代码风格符合项目规范

### 测试策略

#### 单元测试

- 新增工具函数编写单元测试
- 使用 Jest 或 Vitest
- 测试覆盖率目标：新增代码 ≥ 80%

#### 集成测试

- 可视化效果与播放状态同步测试
- 弹窗/面板显示隐藏测试
- 状态持久化测试

#### 性能测试

- 大库场景（5000+ 首）流畅度测试
- 内存占用测试
- 可视化效果性能测试

#### 回归测试

- **核心**：现有三个界面功能完整验证
- 所有现有功能正常工作验证
