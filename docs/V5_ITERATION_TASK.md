## 已做决策

| 决策 | 理由 | 排除的方案及原因 |
|------|------|-----------------|
| 选择 Web Audio API + WebAssembly | 性能极致，支持专业音频处理 | 纯 JavaScript 方案：性能不足 |
| 使用 FFmpeg.wasm | 成熟的无损转换方案 | 自研转换器：开发周期长，稳定性差 |
| Web Workers 后台处理 | 不阻塞主线程，用户体验好 | 主线程处理：大文件会卡顿 |
| 专业/普通双模式 | 兼顾发烧友和普通用户 | 仅专业模式：学习成本高 |

---

## 待解决问题

### 问题 1: FFmpeg.wasm 加载优化
**描述**: FFmpeg.wasm 文件较大，首次加载可能影响性能
**思路**: 
- 懒加载，按需加载
- 使用 CDN 加速
- 提供加载进度提示

### 问题 2: 音频指纹准确率保障
**描述**: 本地音频指纹需要达到 99% 准确率
**思路**:
- 使用成熟的 Chromaprint 算法
- 建立本地指纹数据库
- 多维度匹配（指纹 + 时长 + 文件大小）

### 问题 3: 大库性能优化
**描述**: 10 万首歌曲库需要流畅运行
**思路**:
- IndexedDB 分库存储
- 虚拟列表渲染
- Web Workers 并行处理

---

## 关键约束

1. **100% 纯本地运行**: 所有功能无网络依赖
2. **性能极致**: 10 万首歌曲无卡顿，60fps 稳定
3. **向后兼容**: 基于 V4.0 架构扩展，不破坏现有功能
4. **双模式支持**: 专业/普通模式可切换
