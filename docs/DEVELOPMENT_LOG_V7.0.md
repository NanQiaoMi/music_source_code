# Vibe Music Player V7.0 开发日志

## 📅 版本信息

- **版本**: V7.0
- **日期**: 2026-03-28
- **核心理念**: 离线优先、极致沉浸、高度自定义、零联网依赖
- **技术栈**: Next.js 14 + FastAPI + Zustand + Framer Motion

---

## 🎯 核心定位

基于现有 Next.js+FastAPI 技术栈，迭代开发一款**离线优先、极致沉浸、高度自定义、零联网依赖**的纯本地音乐播放器，核心聚焦本地音乐文件的全生命周期管理、专业级播放能力、沉浸式视觉与交互体验优化。

### 核心原则

- ✅ **全程无任何在线功能**：无在线流媒体、无在线歌词匹配、无云同步、无在线数据上报
- ✅ **完全基于本地文件与本地算力**：所有能力完全基于本地实现
- ✅ **专为本地音乐收藏用户设计**：高自由度、高沉浸感、专业级播放工具
- ✅ **完全复用现有项目体系**：组件体系、状态管理架构、技术栈规范

## 🏗️ 一、分界面精细化迭代开发方案

### 1. 主页（资料库界面）迭代：本地音乐全维度管理中枢

#### 核心开发目标
基于现有 `MusicCardStack`、`LibraryManagerPanel` 核心组件，打造高效、灵活、可视化的本地音乐管理入口，兼顾极简浏览与深度管理能力，与全局视觉体系保持统一，复用现有玻璃拟态 UI 与动态主题能力。

#### 资料库分类与筛选体系升级

##### 拓展基础分类维度
在现有「全部歌曲」基础上，新增 **9 大核心分类**：
- 专辑
- 艺术家
- 流派
- 文件夹
- 我的收藏
- 自定义播放列表
- 最近播放
- 高频播放
- 最近添加

**技术要求**：
- 所有分类基于本地音乐文件元数据自动生成
- 支持用户手动调整分类规则与显示顺序

##### 多维度筛选与排序增强
支持按以下维度进行筛选与排序：
- 音频格式（MP3/FLAC/WAV/APE/ALAC/AAC/OGG/M4A/DSD）
- 码率
- 采样率
- 时长
- 发布年份
- 添加时间
- 播放次数
- 跳过率

**技术要求**：
- 支持多选条件组合筛选
- 筛选结果可直接保存为智能歌单

##### 多视图模式切换
支持 **4 种资料库视图**一键切换：
- 列表视图（带完整元数据）
- 大图网格视图
- 3D 封面流视图
- 时间轴视图

**技术要求**：
- 视图切换复用 Framer Motion 实现顺滑过渡动画
- 适配不同浏览习惯

#### 迷你播放器升级（复用现有浮动播放器核心逻辑）

##### 交互升级
- 支持全局悬浮拖拽
- 点击展开完整 3D 播放器
- hover 呼出快捷操作栏（上一首 / 下一首、收藏、播放模式切换、歌词缩略预览、均衡器快捷入口）

**技术要求**：
- 频谱渲染复用现有 Web Audio API 能力

##### 样式自定义
支持 **3 种迷你播放器样式**一键切换：
- 极简模式（仅播放控制）
- 波形模式（现有频谱样式）
- 封面模式（专辑封面 + 进度条）

**技术要求**：
- 适配不同主页布局
- 样式预设可保存至 `visualSettingsStore`

##### 状态同步
- 播放状态、进度、音量与完整播放器实时同步

#### 核心开发目标

#### 歌词格式与渲染能力升级

- 支持读取音乐文件内嵌歌词（ID3 标签）

- **无任何在线歌词搜索功能**

##### 全维度排版自定义
- 支持歌词字体、字号、字重、行间距、对齐方式（左对齐 / 居中 / 右对齐）自定义
- 支持显示行数（1 行 / 3 行 / 5 行 / 多行滚动）自定义
- 当前播放行支持：
  - 渐变高亮
  - 描边
  - 发光效果
- 非当前行透明度可自由调节
- 支持保存多个歌词样式预设，一键切换

**技术要求**：
- 设置同步至 `lyricSettingsStore`

##### 双语歌词适配
- 支持原文 + 翻译双语歌词同步显示
- 可设置上下排列 / 左右排列
- 翻译行字体、透明度可单独调节

- 修改结果可保存到本地 LRC 文件

##### 歌词操作

  - 收藏歌词片段

- 一键跳转对应歌曲播放

##### 内置歌词编辑器
- 复用 `LyricsCoverEditor` 组件能力
- 新增简易 LRC 歌词编辑功能
- 支持手动制作歌词
- 支持添加时间戳
- 支持修改歌词内容
- 支持双语歌词编辑
- 编辑结果可保存为本地 LRC 文件，或内嵌到音乐文件中

### 4. 可视化界面迭代：本地音频可视化创作与展示中心

#### 核心开发目标
基于现有 Apple Music 风格可视化器核心能力，打造一个独立的、高自由度的音频可视化专属界面，支持丰富的预设效果、深度的参数自定义、用户自制效果导入，实现本地音频与视觉艺术的深度结合，兼具展示性与创作性，基于 WebGL、Web Audio API 实现，与项目技术栈完全兼容。

#### 可视化效果库与分类

##### 内置多类型预设效果
在现有频谱、波形可视化基础上，拓展至至少 **8 大类、30 + 种**内置可视化效果，覆盖不同风格与场景：

**频谱类**：
- 经典柱状频谱
- 环形频谱
- 波浪频谱
- 3D 柱状频谱

**波形类**：
- 动态波形图
- 螺旋波形
- 环形波形
- 多轨波形

**粒子类**：
- 粒子律动
- 粒子爆炸
- 粒子流
- 粒子星云

**几何类**：
- 几何图形律动
- 分形图案
- 万花筒
- 3D 空间几何

**抽象类**：
- 流体模拟
- 色彩流动
- 光效追踪
- 噪点艺术

##### 效果分类与收藏
- 所有效果按类型分组展示
- 支持搜索效果名称
- 支持用户收藏常用效果，一键快速切换
- 支持效果预览，点击效果缩略图即可实时预览当前音频的可视化效果

#### 全参数自定义调节面板

##### 通用参数调节
为所有效果提供基础通用参数调节：

**色彩系统**：
- 支持自定义主色调、辅助色、渐变方案
- 支持从当前专辑封面自动取色（复用 `useDynamicTheme` 能力）
- 支持色彩跟随音频节拍变化

**动效参数**：
- 效果运动速度
- 振幅增益
- 灵敏度
- 旋转角度
- 缩放比例

**布局参数**：
- 效果居中 / 全屏 / 自定义位置
- 元素密度
- 边框样式
- 背景透明度

##### 效果专属参数
针对每一类效果提供专属深度参数：

**粒子类**：
- 粒子数量
- 粒子大小
- 粒子形状
- 粒子生命周期
- 粒子运动轨迹

**频谱类**：
- 频谱柱数量
- 柱体宽度 / 高度
- 柱体圆角
- 柱体排列方式
- 频谱衰减速度

**几何类**：
- 几何图形边数
- 分形迭代次数
- 万花筒镜面数量
- 3D 空间视角

##### 参数调节辅助功能
- 支持参数重置为默认值
- 支持参数保存为「效果预设」（一个可视化效果可保存多组不同参数预设）
- 支持参数实时微调（拖动滑块时可视化效果实时同步变化，无延迟）

**技术要求**：
- 预设保存至 `visualSettingsStore`

#### 自定义可视化效果导入与创作

##### 支持 Shader / 代码导入
- 支持用户导入基于 WebGL/OpenGL 的自定义 Shader 代码
- 支持简单的可视化脚本代码
- 软件提供本地运行环境，实现完全自定义的可视化效果
- 提供代码模板与基础教程入口，降低创作门槛

##### 图片 / 视频素材融合

## 三、技术开发规范与约束（贴合现有项目架构）

### 1. 技术栈复用规范

**前端开发**：
- 必须基于现有技术栈：Next.js 14（App Router）、React 18、TypeScript、Tailwind CSS、Framer Motion、Zustand、Lucide Icons
- 不得引入冗余的第三方框架与库

**音频处理**：
- 必须基于现有 Web Audio API、Web Workers、AudioWorklet 架构
- 复杂音频处理逻辑必须放入 Web Workers 中
- 避免阻塞主线程，保证播放流畅性

**后端开发**：
- 必须基于现有 FastAPI+Uvicorn 架构，Python 3.11+
- 所有本地文件操作、音频格式转换、元数据编辑等能力，均通过后端 API 实现
- API 遵循 RESTful 规范，与前端服务解耦

### 2. 状态管理规范

**Store 拆分原则**：
- 必须复用现有 Zustand 状态管理体系
- 新增功能状态必须拆分至对应 store：
  - UI 状态存入 uiStore
  - 音频播放状态存入 audioStore
  - 播放列表状态存入 playlistStore
  - 歌词设置存入 lyricSettingsStore
  - 视觉设置存入 visualSettingsStore
- 不得新增冗余 store，避免状态耦合

**状态持久化**：
- 全局状态必须支持持久化到本地，刷新页面后状态不丢失
- 所有状态修改必须通过 store 内的 action 实现
- 不得直接修改状态

### 3. 组件开发规范

**组件复用原则**：
- 必须复用现有核心组件，新增功能优先基于现有组件拓展
- 不得重复造轮子
- 新增组件必须按功能拆分，放入对应 src/components 目录下的子文件夹
- 遵循现有项目的组件拆分规范

**类型安全**：
- 组件必须使用 TypeScript 编写
- 定义完整的 Props 类型与接口
- 保证类型安全
- 通用组件必须支持按需引入，避免全局打包冗余

**动画规范**：
- 组件动画必须基于 Framer Motion 实现
- 与全局动效规范保持统一
- 不得引入其他动画库

### 4. 自定义 Hooks 开发规范

**Hooks 封装原则**：
- 新增通用能力必须封装为自定义 Hooks
- 放入 src/hooks 目录
- 遵循现有 Hooks 的命名与开发规范
- 如音频相关能力封装为 useAudioPlayer 拓展、主题相关能力封装为 useDynamicTheme 拓展

**Hooks 质量要求**：
- Hooks 必须遵循 React Hooks 开发规范
- 支持按需调用
- 无副作用泄漏
- 保证性能

### 5. 性能与离线约束

**离线约束**：
- 所有功能必须保证纯本地离线可用
- 无任何联网请求
- 不得引入任何在线 CDN 资源、在线接口、在线服务

- 无卡顿、无加载缓慢问题

**性能策略（大型音乐库）**：

- 调节精度：高精度（1% 精度）

**音频特效**：
- 技术：需要 WebAssembly
- 特效上限：最多 10 个特效同时启用
- 预设：支持用户自定义保存
- A/B 对比：支持

## 🔍 新增需求六轮深度访谈关键决策总结

基于 2026-03-28 的**新增需求六轮深度访谈**，以下是所有新增关键技术和功能决策：

### 新增需求第一轮：可视化录制、响度分析、音频指纹、DLNA 投屏

**可视化录制**：
- 录制技术：MediaRecorder API
- 性能策略：保持原样录制（不降低效果）
- 长音频：分片录制
- 保存策略：先预览再保存

**响度分析**：
- ReplayGain：需要解码完整文件
- 分析策略：后台异步分析
- 应用方式：仅在播放时应用（不修改原文件）
- 手动调整：支持用户手动调整每首歌的响度增益

**音频指纹**：
- 算法库：Chromaprint 编译到 WebAssembly
- 匹配策略：匹配现有库数据
- 指纹库：增量更新
- 降级策略：降级到文件名匹配

**DLNA 投屏**：
- 协议实现：现成 DLNA 库
- 传输策略：实时转码流
- 本地播放：投屏后继续本地播放
- 多设备：支持多设备同时投屏

---

### 新增需求第二轮：推荐算法、重复文件、多显示器、系统托盘

**推荐算法**：
- 算法复杂度：需要预训练模型
- 更新策略：每次播放后增量更新
- 多样性：需要多样性控制，避免重复推荐

**重复文件**：
- 判断策略：自动判断 + 用户手动选择 + 自动删除
- 安全机制：需要回收站机制

**多显示器**：
- 同步策略：状态完全同步
- 异常处理：自动回落到单屏模式
- 控制方式：副屏独立控制
- 配置记忆：记住上次的多显示器配置

**系统托盘**：
- 托盘图标：动态托盘图标（根据播放状态变化）
- 托盘菜单：完整托盘菜单
- 关闭策略：最小化到托盘
- 托盘通知：需要托盘通知（歌曲切换时弹出）

---

### 新增需求第三轮：技术规范、无障碍、数据迁移、Hi-Res 音频

**技术规范**：
- 代码重构：先重构旧代码符合规范
- 第三方库：允许引入必要的新库（如 Chromaprint WASM、FFmpeg WASM）
- 技术验证：先做技术验证（POC）再大规模使用
- TypeScript：必要时允许使用 any 类型

**无障碍适配**：
- ARIA 标签：所有可交互元素都加 ARIA 标签
- 键盘导航：完整键盘导航（方向键 + 快捷键）
- 高对比度：完全重绘（不只是调整颜色）
- 实现时间：无障碍可以后续迭代（V7.0 不必需）

**数据迁移**：
- 迁移脚本：需要自动迁移脚本
- 降级策略：迁移失败时保留旧数据
- 回滚支持：支持从 V7.0 回滚到旧版本
- 触发方式：首次启动自动执行 + 用户手动触发

**Hi-Res 音频**：
- 解码方案：FFmpeg WASM 解码
- 性能预期：FFmpeg 性能足够实时播放
- DSD 播放：DSD 原生播放
- 加载策略：一次性加载到内存（不流式加载）

---

### 新增需求第四轮：大库性能、玻璃拟态、WASM 加载、多语言

**大库性能**：
- 虚拟滚动：react-window
- 搜索策略：后端建立索引
- 数据库：SQLite
- 冷启动：优先显示缓存数据

**玻璃拟态性能**：
- 降级策略：不需要自动降级
- 控制方式：用户手动开关
- 降级程度：只去掉模糊效果（保留玻璃拟态）
- 动效策略：动效保持开启（不禁用）

**WASM 加载**：
- 加载时机：启动时加载所有 WASM
- 降级策略：WASM 失败降级到纯 JS
- 缓存策略：需要缓存 WASM 模块

**多语言**：
- 翻译范围：界面 + 文档都翻译
- i18n 库：i18next
- 切换方式：实时切换（不需要重启）
- 实现时间：V7.0 必须完成

- 失败恢复：自动回滚
- 撤销支持：支持批量内嵌操作的撤销/重做

**CUE 分轨**：
